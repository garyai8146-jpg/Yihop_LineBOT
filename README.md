# 🍲 餐廳清潔照 SOP 自動化管理系統 (LINE Bot Automation)

## 🎯 專案背景與核心價值
本系統為實體火鍋店量身打造，旨在解決傳統餐飲業「清潔查核不易、回報流程繁瑣」的管理痛點。透過將稽核流程數位化，實現零學習成本的 SOP 稽核，**本專案目前已實際投入門市日常營運中。**

## 🛠️ 技術核心與設計決策

### 1. 零儲存成本的「流式影像防偽」 (Zero-Storage Anti-Fraud)
* **核心技術**：攔截 LINE 伺服器傳遞的影像 Byte Stream，在記憶體中即時進行 **MD5 雜湊運算**，不落地儲存實體檔案。
* **開發優勢**：徹底消滅雲端儲存空間 (S3/GCS) 成本與磁碟 I/O 瓶頸，同時迴避資料隱私風險。
* **防呆機制**：透過雜湊值比對，精準阻擋「重複上傳舊照」的作弊行為，確保稽核數據的真實性。

### 2. 輕量化狀態機與多輪對話 (Conversational FSM)
* **交互設計**：實作基於記憶體的有限狀態機 (Finite State Machine)，透過引導式對話（如：`設定 小明 內場`）完成任務初始化。
* **生命週期管理**：基於門市現場行為觀察，採用 Ephemeral State (短暫狀態管理) 模式。任務完成後立即釋放記憶體，確保系統在高頻率使用下仍維持極低資源佔用。

### 3. 執行緒安全與併發處理 (Concurrency Control)
* **Thread-Safety**：針對門市交接班高峰期，導入 `threading.Lock` 確保多名員工同時批次上傳時，計數器與狀態更新具備原子性 (Atomicity)，解決 Race Condition 問題。

### 4. 基礎設施與高可用性設計 (Infrastructure & High Availability)
* **克服冷啟動延遲**：針對 Render 伺服器的休眠機制，導入外部排程監控 (`cron-job.org`) 實作心跳監控 (Keep-alive)，確保系統 24 小時維持熱啟動狀態，達成毫秒級 Webhook 回應速度。
* **DevOps 最佳實踐**：嚴格執行配置與代碼分離，將敏感金鑰抽離至環境變數 (Environment Variables)，確保開發環境與營運環境的安全隔離。

## ⚙️ 技術棧
* **語言與框架**：Python / Flask
* **通訊通訊**：LINE Messaging API SDK
* **維運與部署**：Render (PaaS) / Cron-job.org

## 📈 解決的商業痛點
* **管理自動化**：系統自動清點照片並主動回報缺漏張數，主管無需人工介入核對。
* **營運零成本**：透過流式處理與 In-memory 架構，達成零資料庫、零儲存空間維護費用。
* **高可用性**：基於員工最熟悉的 LINE 介面，門市人員無需安裝額外 App 即可即時上手。
