# 🍲 餐廳清潔 SOP 自動化管理系統 (LINE Bot)

## 🎯 專案背景與核心價值
本系統為解決實體火鍋店（潮州店、內埔店）日常清潔查核的數位化痛點而設計。透過員工最熟悉的 LINE 介面，取代傳統繁瑣的紙本或群組回報，實現高度自動化的 SOP 稽核流程。**本專案目前已實際投入兩間門市的日常營運使用中。**

## 🛠️ 技術核心與設計決策

### 1. 零儲存成本的「流式影像防偽」 (Zero-Storage Anti-Fraud)
* **技術實現**：直接攔截 LINE 伺服器傳遞的影像 Byte Stream，在記憶體中即時進行 **MD5 雜湊運算**。
* **商業價值**：系統不落地儲存任何照片實體，徹底消滅了雲端儲存空間 (S3/GCS) 的昂貴成本與硬碟 I/O 瓶頸。
* **防呆機制**：透過雜湊值比對，精準阻擋「重複上傳舊照」或「一張多傳」的作弊行為，確保稽核真實性。

### 2. 輕量化狀態機與多輪對話 (Conversational FSM)
* **交互設計**：採用極簡指令觸發（例如：`設定 小明 內場`），引導式完成「分店選擇 → 區域確認 → 任務啟動」的多輪對話流程。
* **生命週期管理**：基於員工「設定完即刻上傳」的行為觀察，採用 In-Memory Ephemeral State (短暫狀態管理)，任務完成立即釋放，不佔用長期數據庫資源，確保高併發下的回應速度。

### 3. 執行緒安全與穩定性 (Concurrency & Reliability)
* **Thread-Safety**：導入 `threading.Lock` 處理高峰時段多名員工同時批次上傳的情境，確保計數器與狀態更新的原子性 (Atomicity)，避免遺漏計算。
* **UX 優化**：針對 LINE 伺服器偶發的 410 (Content Gone) 等網路錯誤實作異常攔截，並提供精確的補傳指引。

## ⚙️ 技術棧
* **Backend**: Python / Flask
* **API**: Line Messaging SDK
* **Security**: MD5 Hash Verification, 環境變數 (Environment Variables) 抽離機密資訊
* **Deployment**: Render (PaaS) 搭配外部心跳監控 (Keep-alive) 最佳化冷啟動延遲

## 📈 實質營運成效
* **管理自動化**：系統自動追蹤進度與缺漏張數，主管無需人工逐一清點照片。
* **零維護成本**：基於流式處理架構，系統運行至今維持零資料庫、零儲存空間開銷。
* **快速導入**：門市人員無需下載 App，導入當天即可快速上線使用。
